## 修复 libpng iCCP warning
- 文件：`Market_Watchlist_front_end_local.py`
- 问题：输入股票代码打开 K 线图时，libpng 输出 5 条 "iCCP: known incorrect sRGB profile" warning
- 根因：Windows 上 libpng 的 warning 通过 UCRT (ucrtbase.dll) 的 C 层 stderr 输出，Python 的 os.dup2 / warnings 模块都无法拦截
- 尝试过的无效方案：
  1. `os.dup2` 重定向 fd 2 — Windows 上不影响 C 运行时的 FILE*
  2. PIL 去除 PNG ICC profile — matplotlib PNG 文件本身没有 ICC profile（warning 来源不明）
  3. context manager 临时重定向 — `os.dup2` 在 Windows 上对 C stderr 无效
- 最终修复：程序启动时用 `ctypes.CDLL('ucrtbase.dll')` 调用 `__acrt_iob_func(2)` 获取 C 层 stderr FILE*，再用 `freopen(b'NUL', b'w', stderr_ptr)` 永久重定向到 NUL
- **重要修正**：freopen 会关闭 fd 2，导致 Python `sys.stderr` 底层 FileIO 失效 → yfinance/Flask 写日志时报 I/O 错误 → 股票数据 API 静默失败 → tksheet 表格除 Ticker 和 Market Breadth 外全空
- 修正方案：freopen 前用 `os.dup(2)` 保存控制台 stderr fd，freopen 后用 `os.fdopen(_saved_fd, 'w')` 重建 `sys.stderr` → 控制台
- 整个 block 必须在 `import stock_watch_list_back_end` 之前执行，否则 Flask/yfinance 的 logging handler 已绑定旧 sys.stderr

- 移除：OpenAI SDK 依赖、qwen-plus 模型调用、`BATCH_SIZE` 常量、JSON 解析容错逻辑
- `raw_answer` 字段改为存储爬取原始值（如 "33.88"/"n/a"/"http_403" 等）
- 验证结果：12 只测试标的全部与 StockAnalysis 网站数据一致，SPCX 正确返回 None（网站标注 n/a）
- 文件名保持 `qwen_forward_pe.py` 不变，避免改动后端 import
